        # 内置模块

        • fs：文件系统模块，用于读写文件和文件夹的操作。

        • path：路径模块，用于处理文件路径。

        • http：HTTP 模块，用于创建 HTTP 服务器和客户端。

        • os：操作系统模块，提供操作系统相关的信息和操作。

        • process：进程模块，提供有关进程的信息和操作。

        • events：事件模块，提供事件相关的操作。

          使用内置模块非常简单，只需要使用 require 函数引入即可，例如

          const fs = require('fs')

        # fs

        引入方法： const fs = require('fs')
        参数解析 链接 (opens new window)

        fs.writeFile(file, data[, options], callback)

file <string> | <Buffer> | <URL> | <integer> filename or file descriptor
data <string> | <Buffer> | <TypedArray> | <DataView> | <Object>
options <Object> | <string>
    encoding <string> | <null> Default: 'utf8'
    mode <integer> Default: 0o666
    flag <string> See support of file system flags. Default: 'w'.
    signal <AbortSignal> allows aborting an in-progress writeFile
callback <Function>
    err <Error>

mode	fs.constants.F_OK | fs.constants.W_OK | fs.constants.R_OK  //   是否存在，可写，可读

读文件内容

fs.readFile(__dirname+'/node笔记.md','utf-8',(error,data)=>{})

判断是否为文件夹

fs.stat(filePath,(err,stats)=>{
    console.log(stats.isDirectory());
    console.log(stats);
})

判断该路径是否存在，文件是否存在

判断文件是否可获取
fs.access(path[, mode], callback)
// Check if the file exists in the current directory.
fs.access(file, constants.F_OK, (err) => {
  console.log(`${file} ${err ? 'does not exist' : 'exists'}`);
});

// Check if the file is readable.
fs.access(file, constants.R_OK, (err) => {
  console.log(`${file} ${err ? 'is not readable' : 'is readable'}`);
});

// Check if the file is writable.
fs.access(file, constants.W_OK, (err) => {
  console.log(`${file} ${err ? 'is not writable' : 'is writable'}`);
});

        # path

        获取方法：const path = require("path") path链接 (opens new window)

        方法 作用 实例 结果
        path.basename() 获取文件 path.basename("c:/a/b/c/d.html") d.html
        path.dirname() 获取路径 path.dirname("c:/a/b/c/d.html") c:/a/b/c/
        path.extname() 获取文件后缀名 path.extname("c:/a/b/c/d.html") .html
        path.join() 合并目录 path.join("a","b","c") a\b\c
        path.resolve() 解析目录 path.resolve("a","b","c") yourDir\a\b\c
        path.resolve("/a","b","c") C:\a\b\c
        path.resolve("a","../b","c") yourDir\b\c
        path
const pathStr = path.join('/a', '/b/c', '../../', './d', 'e')
console.log(pathStr)  // \a\b\d\e

const fpath = '/a/b/c/index.html'

const fullName = path.basename(fpath)
console.log(fullName)   // 输出index.html

const nameWithoutExt = path.basename(fpath, '.html')
console.log(nameWithoutExt)   // 输出index

const fext = path.extname(fpath)
console.log(fext)  // 输出.html

        # http

        使用 const http = require("http") http链接 (opens new window)

        request属性or方法 含义 response属性or方法 含义
        request.aborted 请求取消的话这个值为 true
        request.getHeader(name) 获取请求头的某个属性，request.getHeader('Content-Length') response.getHeader(name) 获取响应头的某个属性，response.getHeader('Content-Length')
        request.path 请求路径
        request.method 请求方法
        request.host 请求域名
        request.protocol 请求协议
        request.removeHeader(name) 移出某个请求头，request.removeHeader('Content-Type') response.removeHeader(name) 移出某个响应头，response.removeHeader('Content-Type')
        request.removeHeader(name) 移出某个请求头，request.removeHeader('Content-Type')
        request.setHeader(name,value) 设置请求头request.setHeader('Cookie', ['type=ninja', 'language=javascript']); response.setHeader(name,value) 设置响应头response.setHeader('Content-Type', 'text/html');;
        http
// 导入
const http = require("http");

// 创建web服务器实例
const server = http.createServer();

// 为服务器绑定request事件
server.on("request",(req,res)=>{
    const url = req.url;
    const { query } = url.parse(req.url,true); // 解析get方法的查询的参数，query是一个对象
    res.setHeader("Content-Type","text/html;charset=utf-8");
    //const str = `Your request url is ${req.url},and request method is ${req.method}`;
    if(url === "/" || url ==="/index.html"){
        res.end("This is an index html")
    }else if(url === "/about.html"){
        res.end("<h1>About Page</h1>")
    }
})

// 启动服务器  监听80端口
server.listen(80,()=>{
    console.log("127.0.0.1:80 is running");
})
